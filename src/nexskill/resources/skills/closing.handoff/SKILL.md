# Handoff and Closeout

Produce a concise closeout an owner can act on.

## When to use

Use this skill at the **closing** stage, after verification.

## Process

1. State what changed in feature or function terms, not file-by-file.
2. State what was verified and where it was pushed.
3. State what remains and the recommended next action.
4. Exclude secrets, raw transcripts, and any provider or source names.

## Output

A closeout report that records what passed, what remains, and the next
recommended action. The owner or project process decides what happens next.
